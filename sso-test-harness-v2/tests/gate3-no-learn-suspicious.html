Without this protection, an attacker could:
  1. Lure user to phishing page once
  2. PhishWatch would "learn" attacker's exfil domain
  3. Future attacks from same page would be silently allowed
Solution: Only learn from benign windows (no mismatch, no new origins).

Step 1: Trigger Auth Window

Focus on password field to arm the treadmill

Step 2: Simulate Suspicious Activity

This simulates cross-origin network activity to an unknown destination

Instructions:
  1. Open DevTools Console (F12)
  2. Filter by [PhishWatch
  3. Click on the password field (Step 1)
  4. Quickly click "Simulate Suspicious POST" (within 2 seconds)
  5. Wait for treadmill to disarm
  6. Verify you see:
    • form_action_origin_mismatch signal (form action is suspicious)
    • novelty: NOT learning origins (suspicious indicators)
    • hasMismatch: true OR hasNewOrigin: true
  7. Click "Check Storage" - attacker origin should NOT be learned

Critical: The attacker origin (https://evil-exfil.attacker.test) should NOT appear in learned origins.
