✅ Gate 2 Test: Same-Site Form Action

SHOULD NOT FIRE Form action on same registrable domain is NOT suspicious

ℹ️ Same-site scenario
Page origin: http://localhost:8080 (simulating app.example.com)
Form action: http://localhost:8080/auth/login (simulating auth.example.com)

In real SSO, this would be like app.company.com posting to login.company.com

Company Login

Form posts to auth subdomain (same registrable domain)

Instructions:
  1. Open DevTools Console (F12)
  2. Filter by [PhishWatch
  3. Click on the password field
  4. Verify you see:
    • form action analysis... originMismatch: false (same origin in test)
    • OR sameSite: true, isSuspicious: false if testing with real subdomains
    • NO form_action_origin_mismatch signal

Why: Many legitimate auth flows post to different subdomains within the same organization. This is normal and should not trigger mismatch detection.

To test with real subdomains:

  1. Add to /etc/hosts:
    127.0.0.1 app.testsite.local
    127.0.0.1 auth.testsite.local
  2. Access page at http://app.testsite.local:8080/
  3. Set form action to http://auth.testsite.local:8080/login
